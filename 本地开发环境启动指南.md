## 📋 前置准备

### 1. 确认本地服务已安装

- [x] **MySQL 8.0+** (端口 3306, 用户: root, 密码: 123456)
- [x] **Redis** (端口 6379)
- [x] **MinIO** (端口 9000, 用户: minioadmin, 密码: minioadmin)

### Python 环境

- [x] **Python 3.11+**

怎么安装 Python 3.11+

- **macOS**：使用 Homebrew 安装

```bash
brew install python@3.11
```

- **Windows**：从 [Python 官方网站](https://www.python.org/downloads/windows/) 下载并安装

怎么确认 Python 3.11+ 安装成功

```bash
python3.11 --version
```

怎么确认最新的 Python3 版本

```bash
python3 --version
```

### 2. 安装 uv (如未安装，Java 版可以跳过)

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

**uv** 是一个用于管理 Python 包的工具，类似于 **pip**。安装完成后，你可以使用 **uv** 来安装和管理 Python 包。如果已经安装，可跳过。

### 前端环境

- [x] **Node.js 18+** (推荐使用 [nvm](https://github.com/nvm-sh/nvm) 管理)
- [x] **npm** (Node.js 包管理器)

怎么安装 Node.js 18+

- **macOS**：使用 Homebrew 安装

```bash
brew install node@18
```

- **Windows**：从 [Node.js 官方网站](https://nodejs.org/en/download/) 下载并安装

怎么确认 Node.js 18+ 安装成功

```bash
node --version
```

怎么确认最新的 Node.js 版本

```bash
node -v
```

如何安装 npm

- **macOS**：使用 Homebrew 安装

```bash
brew install npm
```

- **Windows**：npm 会自动安装在 Node.js 中

怎么确认 npm 安装成功

```bash
npm --version
```

什么是 npm

npm 是 Node.js 包管理器，用于安装和管理 Node.js 模块。它是 Node.js 平台的默认包管理器，用于安装和管理 Node.js 模块。

pnpm 呢？

pnpm 是 Node.js 包管理器，用于安装和管理 Node.js 模块。它是 Node.js 平台的默认包管理器，用于安装和管理 Node.js 模块。

和 npm 有什么区别？

- **npm**：每个项目都有自己的 `node_modules` 目录，安装的包会被复制到项目目录中。
- **pnpm**：使用符号链接（symlink）和硬链接（hard link）来共享依赖，避免了 npm 中的重复安装问题。

## 🚀 启动步骤

### 第一步: 启动本地基础设施服务

#### 1.1 启动 MySQL

```bash
# macOS (使用 Homebrew)
brew services start mysql

# 或手动启动
mysql.server start

# 验证
mysql -h localhost -P 3306 -u root -p123456 -e "SELECT 1"
```

如果已经启动，忽略。

首次启动项目时，请根据脚本 [init-local-mysql.sh](scripts/init-local-mysql.sh) 来完成库表初始化，正常我们会创建下面几个库(也可以直接使用 [docker/astronAgent/mysql](docker/astronAgent/mysql) 下的sql文件进行手动初始化)

- agent.sql -> 对应库 paiflow-agent ---> python core/agent微服务使用的库（以工作流的视角，这个非强依赖）
- link.sql -> 对应库 paiflow-link ---> python plugin/link微服务使用的库
- schema.sql -> 对应库 paiflow-console  ---> hub应用主要使用的库
- tenant.sql -> 对应库 paiflow-tenant  ---> python tenant微服务使用的库
- workflow.sql -> 对应库 paiflow-workflow  --> python workflow/core-workflow-java 两个工作流使用的库

#### 1.2 启动 Redis

```bash
# macOS (使用 Homebrew)
brew services start redis

# 或手动启动
redis-server

# 验证
redis-cli -h localhost -p 6379 ping
# 应该返回: PONG
```

如果已经启动，忽略。

#### 1.3 启动 MinIO

```bash
# macOS (使用 Homebrew)
brew services start minio

# 或手动启动
./minio server data/

# 验证
curl http://localhost:9000/minio/health/live
# 应该返回 200 OK
```

**MinIO 初始化** (首次运行):

1. 访问 http://localhost:9000
2. 登录: minioadmin / minioadmin
3. 创建 Bucket: `console`, `workflow`, `aitools`（需要 public）

## 🐛 调试技巧

### Console Hub 调试

1. 在 IDEA 中打开 Java 代码 (如: `WorkflowChatController.java`)
2. 在方法中**设置断点**
3. 以 **Debug 模式**启动 Console Hub
4. 通过前端或 API 工具触发请求
5. 断点命中,可以调试 Java 代码

### 前端调试

1. 打开 Chrome 浏览器
2. 按 `F12` 打开 DevTools
3. 在 `Sources` 面板中找到源代码
4. 设置断点,刷新页面即可调试
5. 支持热重载,修改代码自动刷新

## 🔧 常见问题

### Q1: 端口被占用怎么办?

```bash
# 查看端口占用
lsof -i :3000   # 前端
lsof -i :8080   # Console Hub
lsof -i :17870  # Agent
lsof -i :7880   # Workflow
lsof -i :18888  # Link
lsof -i :18668  # AITools

# 杀死进程
kill -9 $(lsof -ti :3000)
```

### Q2: MySQL 连接失败?

```bash
# 检查 MySQL 是否启动
mysql.server status

# 检查密码是否正确
mysql -h localhost -P 3306 -u root -p123456 -e "SELECT 1"

# 检查数据库是否存在
mysql -h localhost -P 3306 -u root -p123456 -e "SHOW DATABASES"
```

### Q3: Python 服务启动报错?

```bash
# 检查配置文件是否存在
ls -la core/agent/config.env
ls -la core/workflow/config.env
ls -la core/plugin/link/config.env
ls -la core/plugin/aitools/config.env

# 重新生成配置
./scripts/setup-python-local-debug.sh

# 检查 Python 版本
python3 --version  # 应该是 3.11+

# 检查 uv 是否安装
uv --version
```

### Q4: Console Hub 启动报错?

**检查 application-local.yml**:

```bash
cat console/backend/hub/src/main/resources/application-local.yml
```

确保包含正确的配置 (MySQL, Redis, MinIO)

**检查 Active profiles**:

在 IDEA 运行配置中,确保 `Active profiles` 设置为 `local`

### Q5: 前端无法连接后端?

**检查前端配置**:

```bash
cat console/frontend/.env.development
```

应该包含:
```
VITE_BASE_URL=http://localhost:8080
```

**检查 vite.config.js 代理配置**:

```bash
cat console/frontend/vite.config.js
```

确保 proxy 配置正确

### Q6: MinIO 无法访问?

```bash
# 检查 MinIO 是否启动
curl http://localhost:9000/minio/health/live

# 访问控制台创建 Bucket
open http://localhost:9001

# 登录: minioadmin / minioadmin
# 创建 Bucket: console, workflow, aitools
```
